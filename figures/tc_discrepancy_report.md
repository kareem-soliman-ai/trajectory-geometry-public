# Commitment Timing Discrepancy Report

## Local data sources
- `unified_metrics.csv` stores `time_to_commit`, inherited from the archive metric definition
  `compute_time_to_commit(h, window=6, stride=2)`, which returns the center position of the
  window with the largest radius-of-gyration drop.
- `analysis_C_window_metrics.csv` was generated by `04_analysis_C_token_level_dynamics_sliding_windows.py`
  with `window=5`, `stride=1`, at layer 16, on a sampled subset of trajectories (10 per group,
  except 4 G1 examples in the saved artefact).

## Observed discrepancy
The manuscript text currently describes `t_c` as if it were a short controlled-window token count
(`Direct ~ 5`, `CoT ~ 11-14`). The local archived metric behaves differently.

## Correct local summary from `unified_metrics.csv`
- Direct (all available layer-0 rows): mean 14.32, median 17.0, std 6.28
- CoT (all available layer-0 rows): mean 40.01, median 39.0, std 16.84
- G2 only: mean 12.46, median 14.0, std 5.97
- G4 only: mean 40.26, median 41.0, std 13.78

## Interpretation
The local archive supports the qualitative claim that Direct trajectories commit earlier than CoT
trajectories, but it does not support the manuscript's current `5` versus `11-14` wording.
The archived `time_to_commit` should be described as a trajectory-index measure derived from a
`window=6`, `stride=2` radius-of-gyration drop analysis, not as a short controlled-window token
count from `analysis_C_window_metrics.csv`.
